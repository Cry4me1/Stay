# Stay

> 让网页陪你待着。

Stay 是一个极简的环境音 + 专注网页项目。它不追踪、不分析、不提醒、不优化效率。它只提供一种**可共处的数字存在**：当你工作、发呆或熬夜时，网页在背景中安静地呼吸。

---

## 一句话定义

**这是一个什么都不要求你的网页。**

---

## 核心理念

* **陪伴而非工具**：不提供任务、目标或反馈机制
* **低干扰优先**：任何元素都必须接近感知阈值
* **克制即设计**：知道什么不做，比做什么更重要
* **可消失的界面**：UI 永远不应成为注意力中心

---

## 功能范围（刻意收敛）

### 1. 环境音（Environment Sound)

仅提供三种无旋律、无节拍的环境音：

* 🌧 Rain —— 连续、均匀、可无限循环
* ☕ Cafe —— 模糊人声与器具声的低频混合
* 🌬 Wind —— 缓慢起伏的宽频噪声

设计原则：

* 不传递情绪高潮
* 不引导节奏
* 长时间播放不疲劳

---

### 2. 声音驱动的视觉层（Audio-driven Visual)

视觉不是独立动画，而是**声音的副产物**。

* 页面背景为渐变 / 雾状 / 噪声纹理
* 通过音频低频能量驱动：

  * 背景亮度微幅变化
  * 渐变位置缓慢漂移
  * 纹理密度轻微呼吸

用户在大多数情况下：

> *“感觉它在动，但说不清哪里在动。”*

---

### 3. UI 覆盖层（可完全隐藏）

UI 只在必要时出现。

* 默认极简：环境选择 + 音量
* 鼠标静止数秒后自动淡出
* 支持手动隐藏（快捷键 / 手势）
* 完全隐藏后：

  * 页面仅剩声音与背景

UI 是访客，不是常驻者。

---

## 技术实现概览

### 技术栈

* 原生 HTML / CSS / JavaScript
* Web Audio API
* Canvas 2D 或 CSS Variables
* 无框架、无构建工具（初期）

---

## 交互与状态设计

### UI 状态

* `visible`：用户刚交互
* `fading`：准备消失
* `hidden`：完全不占用注意力

状态切换原则：

* 自动行为永远低于用户主动行为
* 不因偶然鼠标抖动频繁切换

---

## 明确不做的事情

以下内容**刻意不进入项目范围**：

* ❌ 播放进度条
* ❌ 时间 / 番茄钟 / 统计
* ❌ 登录、账户、云同步
* ❌ 个性化推荐
* ❌ 明确的“效率提升”承诺

理由：

> 一旦开始衡量，人就不再待着，而是在使用工具。

---

## MVP 定义

**打开网页 → 选择一个环境音 → UI 消失 → 页面陪你待着。**

如果实现超出这句话，说明已经开始偏离。

---

## 发布标准

当以下条件全部满足时，即可发布：

* 可连续运行数小时无明显疲劳
* UI 在隐藏状态下无存在感
* 视觉变化不影响阅读或思考
* 项目没有“继续加功能的冲动”

---

## 项目态度声明

Stay 不试图解决问题。

它只是承认：

> 有时候，人只是想有人在场。
